# 时间轴轨道系统 功能设计文档

---

## 一、产品背景与目标

**现状问题**
当前版本的底图和音频以"全局配置"形式存在于左侧面板，意味着整段视频只能使用一张底图和一段音频，无法表达时间维度的素材变化。

**新版目标**
引入三轨时间轴，允许用户在时间维度上编排底图/视频素材与音频，同时将字幕以可视化片段的形式呈现在轨道上，形成直观的"所见即所得"剪辑体验。

**设计边界**
本应用定位为字幕生成工具而非专业剪辑软件，仅支持固定三条轨道，无多轨混合、转场特效、音频剪辑等专业功能。

---

## 二、整体布局变更

### 2.1 新布局结构

```
┌──────────────┬──────────────────────────────┬──────────────┐
│              │                              │              │
│   左侧面板   │        画布预览区            │  右侧编辑面板 │
│   (精简后)   │                              │  (不变)      │
│              │                              │              │
├──────────────┴──────────────────────────────┴──────────────┤
│  时间轴区域（新增，固定高度约 200px，可拖拽上边缘调整高度）  │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 时间轴区域高度
- 默认高度：200px
- 拖拽上边缘可调整，范围：140px – 320px
- 折叠按钮（「∧」）可将轨道区折叠至仅显示标题栏（32px）

## 三、左侧面板变更

移除以下两个 section（功能转移至时间轴轨道）：

| 移除项 | 新入口 |
|--------|--------|
| 背景（纯色 / 图片 / 调节） | 媒体轨道 |
| 音乐（上传 / 清除） | 音频轨道 |

**保留项：** LRC 歌词输入、随机特效开关、画布尺寸、字幕样式、导出

> **注意**：纯色背景配置保留在左侧面板"字幕样式"内，作为无媒体素材时的兜底背景色。媒体轨道有内容时，纯色背景被媒体层遮挡；无内容时显示纯色背景。

## 四、时间轴区域详细设计

### 4.1 整体结构

```
┌─────────────────────────────────────────────────────────────┐
│  [时间标尺]  0s    2s    4s    6s    8s    10s  ...          │
├──────┬──────────────────────────────────────────────────────┤
│  🖼  │  [图片片段]  [视频片段]         [图片片段]            │  ← 媒体轨道
├──────┼──────────────────────────────────────────────────────┤
│  💬  │  [字幕A]  [字幕B]    [字幕C]        [字幕D]          │  ← 字幕轨道
├──────┼──────────────────────────────────────────────────────┤
│  🎵  │  [━━━━━━━━━━━━━━━━━音频片段━━━━━━━━━━━━━━━━]         │  ← 音频轨道
└──────┴──────────────────────────────────────────────────────┘
        ↑ 播放头（红色竖线，可拖拽）
```

- 左侧固定宽度约 48px 的轨道标签区，显示图标和轨道名
- 右侧为可横向滚动的内容区

---

### 4.2 媒体轨道

#### 功能描述
支持导入任意数量的图片和视频素材，按时间顺序排列，顺序播放。片段之间不允许时间重叠。

#### 片段类型

| 类型 | 支持格式 | 特性 |
|------|----------|------|
| 图片 | JPG / PNG / WebP | 默认时长 5s，可拖拽右边缘调整 |
| 视频 | MP4 / WebM | 时长由视频文件决定，不可缩短超出源时长 |

#### 添加素材
- **拖拽导入**：将文件拖拽到媒体轨道区域，自动追加到末尾
- **点击添加按钮**：轨道标签右侧有「+」按钮，点击弹出文件选择器，支持多选
- 新片段总是追加到最后一个片段之后（不支持任意插入间隙，保持简单）

#### 片段外观
- 图片片段：缩略图填充，右下角显示时长
- 视频片段：首帧缩略图 + 视频图标角标，右下角显示时长
- 片段名称截断显示在顶部

#### 片段交互

| 操作 | 交互方式 |
|------|----------|
| 选中片段 | 单击，选中后显示蓝色边框 |
| 调整图片时长 | 拖拽右边缘（仅图片可拖，视频不可缩短） |
| 删除片段 | 选中后按 Delete 键，或右键菜单「删除」 |
| 重排顺序 | 拖拽片段左右移动，其他片段自动平移让位 |
| 预览画面 | hover 时弹出小预览气泡（图片：原图；视频：首帧） |

#### 图片亮度 / 对比度 / 饱和度调节
选中图片片段后，右侧出现属性面板（复用现有样式），显示亮度 / 对比度 / 饱和度滑条。

---

### 4.3 字幕轨道

#### 功能描述
字幕轨道内容**由 LRC 自动生成**，每条歌词对应一个不可移动的片段（位置由 LRC 时间戳决定）。点击片段可在右侧面板编辑该行字幕的所有属性。

#### 片段外观
- 背景色：半透明紫色（与应用 accent 色一致）
- 显示文字内容（超长截断，hover 显示 tooltip）
- 宽度 = 该行字幕的持续时长

#### 字幕轨道交互

| 操作 | 交互方式 |
|------|----------|
| 选中字幕 | 单击片段，同步选中右侧编辑面板该行 |
| 跳转预览 | 单击字幕片段，播放头跳转到该字幕开始时间 |
| 编辑属性 | 选中后在右侧面板修改（与现有逐行编辑器相同） |
| 重排 / 移动 | **不支持**（时间由 LRC 决定） |
| 删除 | **不支持**（通过修改 LRC 重新生成） |

> 字幕轨道是只读的时间映射，所有内容以 LRC 为唯一数据源。

---

### 4.4 音频轨道

#### 功能描述
单条音频轨道，支持多个音频文件，按时间顺序播放。

#### 添加音频
- 轨道标签右侧「+」按钮，点击弹出文件选择器
- 支持格式：MP3 / AAC / WAV / OGG
- 添加后显示为一条贯穿轨道的波形色块（不渲染真实波形，使用颜色渐变条代替）
- 片段之间不允许重叠
- 右侧显示文件名和总时长

#### 音频轨道交互

| 操作 | 交互方式 |
|------|----------|
| 添加音频 | 拖拽多个文件导入，自动追加到轨道末尾；或点击「+」按钮弹出文件选择器，支持多选 |
| 删除音频 | 右键菜单「删除」，或轨道内 hover 时显示的「✕」按钮 |
| 音量 | **暂不支持**（预览播放使用 Audio 元素默认音量） |

---

### 4.5 时间标尺与播放头

#### 时间标尺
- 固定在轨道区顶部，高度 24px
- 根据当前缩放级别动态显示刻度（如 0.5s / 1s / 2s / 5s / 10s 间隔）
- 刻度线细、整秒标粗
- 单击标尺任意位置：播放头跳转到该时间

#### 播放头
- 红色竖线，贯穿全部三条轨道
- 顶部有倒三角拖拽手柄
- 拖拽可以 scrub 时间
- 与上方画布预览区的 seek bar 双向联动

#### 总时长计算

```
总时长 = max(媒体轨道末尾时间, 字幕轨道末尾时间, 音频轨道末尾时间)
```

---

### 4.6 缩放与滚动

| 操作 | 交互方式 |
|------|----------|
| 横向滚动 | 鼠标滚轮（在轨道内容区） |
| 缩放时间轴 | Ctrl + 滚轮（或触控板双指捏合） |
| 缩放级别 | 5 档，从 0.5x（1px = 0.2s）到 8x（1px = 0.03s） |
| 缩放后播放头居中 | 缩放时保持播放头在视口中心 |

## 五、渲染与播放逻辑

### 5.1 帧渲染顺序

```
每帧绘制顺序（由下到上）：
1. 纯色背景（兜底）
2. 当前时间对应的媒体素材帧（图片 or 视频当前帧）
3. 字幕层（按 LRC 时间戳决定可见字幕）
```

### 5.2 视频素材播放
- 每个视频片段对应一个隐藏的 `<video>` 元素（预加载）
- 播放时 seek 到片段内相对时间，通过 `ctx.drawImage(videoEl, ...)` 逐帧绘制
- 导出时逐帧捕获（与现有 WebM 导出逻辑相同）

### 5.3 图片素材显示
- 在该图片片段时间范围内绘制对应的 `HTMLImageElement`
- 应用亮度 / 对比度 / 饱和度调节（与现有图片背景逻辑相同）

---

## 六、导出变更

导出流程无重大变化，仍为逐帧渲染到 canvas 再录制。差异点：

- 渲染时按当前帧时间决定绘制哪个媒体片段（图片 / 视频帧）
- 音频合并：WebM 导出时将音频轨道的 MediaStream 混入录制流（通过 `AudioContext` 混音）
- MOV 导出流程不变（先渲染 WebM，再 ffmpeg 转换）

> **已知限制**：视频素材在导出时通过逐帧 drawImage 捕获，导出速度取决于视频解码性能，非实时。

## 七、实现优先级

| 阶段 | 内容 |
|------|------|
| **P0** | 时间轴区域 UI 框架、时间标尺、播放头、轨道布局 |
| **P0** | 左侧面板移除音频 / 背景 section |
| **P1** | 媒体轨道：图片素材导入、拖拽排序、时长调整、渲染集成 |
| **P1** | 音频轨道：导入、播放联动 |
| **P1** | 字幕轨道：LRC 自动填充、点击选中联动右侧编辑器 |
| **P2** | 媒体轨道：视频素材支持 |
| **P2** | 时间轴缩放、横向滚动 |
| **P2** | 图片素材属性面板（亮度 / 对比度 / 饱和度） |
| **P3** | 导出时音频混入（AudioContext 混音） |
| **P3** | 时间轴区域高度拖拽 / 折叠 |

---

## 八、暂不纳入的方向（留备忘）

- **转场特效**（淡入淡出、溶解）——媒体片段之间的过渡效果，需额外渲染逻辑
- **多音频轨道 / 音量包络**——超出本应用定位
- **视频素材裁剪**（trim in / out）——暂只支持完整播放
- **字幕轨道手动调整时间**——打破 LRC 作为单一数据源的约定
