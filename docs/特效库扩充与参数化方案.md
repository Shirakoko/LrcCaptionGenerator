## 一、现状梳理

### 1.1 现有特效

| 类型 | 数量 | 名称 |
|------|------|------|
| 入场 | 10 | typewriter / slideLeft / slideRight / slideUp / slideDown / scalePop / scatter / flipX / blurFade / wave |
| 持续 | 4  | float / charJitter / breathe / none |

## 三、参数体系设计

### 3.1 设计原则

- **通用扁平结构**：所有特效共享同一个 `EffectParams` 接口，各特效只使用自己关心的字段，忽略其余字段。避免每个特效定义独立类型导致 `EffectOverride` 变成复杂联合类型。
- **默认值注册表**：用一个静态 `EFFECT_PARAM_DEFS` 描述每个特效支持哪些参数及其范围和默认值，UI 直接读取它动态生成控件。
- **向后兼容**：不带参数的旧 override 继续生效，效果函数内部 `params?.field ?? DEFAULT` 降级处理。

### 3.2 `EffectParams` 接口

```typescript
export interface EffectParams {
  // 时序
  duration?:     number;  // 单次动画时长（秒）
  stagger?:      number;  // 逐字延迟（秒）
  // 位移
  distance?:     number;  // 滑动 / 下落 / 上浮距离（px）
  // 缩放
  scale?:        number;  // 目标缩放（zoomIn 起点 scale）
  // 旋转
  angle?:        number;  // 旋转角度（sway 摆幅, rotateIn 初始角度）
  // 模糊
  blurAmount?:   number;  // blur 起点值（px）
  // 散射
  rangeX?:       number;  // scatter / explode X 范围（px）
  rangeY?:       number;  // scatter / explode Y 范围（px）
  // 持续特效
  amplitude?:    number;  // 振幅（px）
  period?:       number;  // 周期（秒）
  interval?:     number;  // glitch 触发间隔（秒）
  // 其他
  spread?:       number;  // dissolve 最大延迟（秒）
}
```

### 3.3 `EffectOverride` 变更

```typescript
// 现在
export interface EffectOverride {
  entrance?: EntranceName;
  idle?:     IdleName;
  exit?:     ExitName;
}

// 新增 params 字段
export interface EffectOverride {
  entrance?:       EntranceName;
  entranceParams?: EffectParams;
  idle?:           IdleName;
  idleParams?:     EffectParams;
  exit?:           ExitName;
  exitParams?:     EffectParams;
}
```

### 3.4 参数定义注册表

```typescript
export interface ParamDef {
  key:     keyof EffectParams;
  label:   string;
  min:     number;
  max:     number;
  step:    number;
  default: number;
  unit:    string;
}

export const ENTRANCE_PARAM_DEFS: Partial<Record<EntranceName, ParamDef[]>> = {
  typewriter:  [
    { key:'duration', label:'字符时长', min:0.02, max:0.3,  step:0.01, default:0.06, unit:'s'  },
    { key:'stagger',  label:'字符间隔', min:0,    max:0.2,  step:0.01, default:0.08, unit:'s'  },
  ],
  slideLeft:   [...],   // distance / duration / stagger
  slideRight:  [...],
  slideUp:     [...],
  slideDown:   [...],
  scalePop:    [...],   // duration / stagger
  scatter:     [...],   // rangeX / rangeY / duration / spread
  flipX:       [...],   // duration / stagger
  blurFade:    [...],   // blurAmount / duration / stagger
  wave:        [...],   // distance / duration / stagger
  fadeIn:      [...],
  zoomIn:      [...],   // scale / duration / stagger
  dropIn:      [...],   // distance / duration / stagger
  rotateIn:    [...],   // duration / stagger
  rise:        [...],   // distance / duration
};

export const IDLE_PARAM_DEFS: Partial<Record<IdleName, ParamDef[]>> = { ... };
export const EXIT_PARAM_DEFS:  Partial<Record<ExitName,  ParamDef[]>> = { ... };
```

## 五、UI 交互设计

### 5.1 逐行编辑面板「特效」区结构变化

```
特效
├── 入场  [select: slideLeft ▾]
│   ├── 位移距离  ──────●──── 80 px
│   ├── 持续时长  ────●────── 0.55 s
│   └── 字符间隔  ●────────── 0.04 s
├── 持续  [select: float ▾]
│   ├── 浮动幅度  ──────●──── 6 px
│   └── 周期      ──────●──── 2.5 s
└── 退场  [select: fadeOut ▾]
    └── 持续时长  ──────●──── 0.45 s
```

**规则**：
- 切换特效类型时，参数控件区域即时重绘（只显示该特效有意义的参数）。
- 参数修改后**不立即提交**，需点击"应用并预览"才写入 override。这与现有布局参数的交互模式一致。
- 若某特效无可调参数（如 `none`），参数区域隐藏。

### 5.2 参数控件复用

参数区使用与布局区完全相同的 `_sliderRow(label, key, value, min, max, step, unit)` 组件，`data-key` 命名前缀区分层级，例如：`entrance-duration`、`idle-amplitude`、`exit-distance`。

### 5.3 _collectOverride 变更

```typescript
// 伪代码
function _collectEntranceParams(panel: HTMLElement): EffectParams {
  const entrance = selectVal('entrance');
  const defs = ENTRANCE_PARAM_DEFS[entrance] ?? [];
  return Object.fromEntries(
    defs.map(d => [d.key, sliderVal(`entrance-${d.key}`)])
  );
}
```

## 六、实现路径

### Phase 1：数据结构 + 参数透传

- [ ] `src/effects/types.ts`：添加 `EffectParams`、`ParamDef`、`ENTRANCE_PARAM_DEFS` 等；`EffectOverride` 加入 `entranceParams? / idleParams? / exitParams?`
- [ ] `src/effects/entrance.ts`：`buildEntrance` 签名加 `params?: EffectParams`，所有硬编码值改为 `params?.xxx ?? DEFAULT`

- [ ] `src/effects/exit.ts`：同上
- [ ] `src/renderer/sceneController.ts`：`_buildTimeline` 中把 `override?.entranceParams` 等传入 build 函数

### Phase 2：新增特效

- [ ] 入场：fadeIn / zoomIn / dropIn / rotateIn / rise
- [ ] 持续：shake / sway / glitch
- [ ] 退场：slideOutLeft / slideOutRight / blurOut / dissolve

### Phase 3：UI 参数控件

- [ ] `lineEditor.ts`：`_buildEffectSection` 方法，根据当前选择的特效名从注册表读取 `ParamDef[]`，动态生成 slider 行

## 七、风险与注意事项

| 风险 | 说明 | 应对 |
|------|------|------|
| RNG 偏移 | 新增特效参数化后，默认参数不调 rng，不引入偏移 | EffectParams 默认值在函数内硬编码，不走 rng |
| `scatter` / `glitch` 类使用 rng | 这类特效仍传入 rng，但 rangeX/Y 可覆盖 | 参数覆盖范围，rng 只决定具体随机点位 |
| idle 时长由 scene 计算 | `buildIdleTween` 的 `duration` 由歌词时长决定，不可覆盖 | `idleParams.duration` 指周期，非总时长 |
| 向后兼容 | 已保存的 override 无 params 字段 | 所有 params 字段可选，效果函数内 `?? DEFAULT` 兜底 |
