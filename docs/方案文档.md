# 动态字幕视频生成器 — 方案文档

## 一、产品概述

一个纯前端 Web 应用，用户上传 `.lrc` 歌词文件，配置参数后，生成带有动态文字特效的**透明底字幕视频**，风格参考手书 PV / 日系 MV 字幕。无需服务器，部署成本极低。

---

## 二、功能需求

### 2.1 输入参数

| 参数 | 说明 |
|------|------|
| `.lrc` 文件 | 标准 LRC 格式，支持逐行时间戳 |
| 分辨率 | 自定义宽×高，预设 1920×1080 / 1280×720 / 竖屏 1080×1920 |
| 字体 | 上传本地字体文件（TTF/OTF）或从预设列表选择 |
| 字色 | 主色 + 描边色，支持渐变色 |
| 字号 | 基础字号，动效会在此基础上做随机缩放 |
| 随机种子 | 可选，固定种子可复现同一结果 |

### 2.2 输出格式

应用提供两个导出按钮，均无需用户安装任何软件：

**① 导出 WebM（VP9 + Alpha）**

- 零依赖，秒级下载
- 适合剪映桌面版、DaVinci Resolve 等直接支持 WebM Alpha 的软件

**② 转换为 MOV（ProRes 4444）**

- 点击后在浏览器内通过 **ffmpeg.wasm** 完成转换，无需安装 ffmpeg
- 适合 Premiere、Final Cut Pro、剪映手机版等不支持 WebM Alpha 的软件
- ffmpeg.wasm 约 30MB，**懒加载**：只有点击此按钮时才下载，不影响主流程

### 2.3 各软件 WebM Alpha 兼容性

| 软件 | WebM Alpha 支持 | 建议导出格式 |
|------|----------------|-------------|
| 剪映（桌面版） | 支持 | WebM 直接导入 |
| DaVinci Resolve | 支持 | WebM 直接导入 |
| Premiere Pro | 不稳定 | MOV ProRes 4444 |
| Final Cut Pro | 不支持 | MOV ProRes 4444 |
| 剪映（手机版） | 不支持 | MOV ProRes 4444 |

### 2.4 透明视频格式对比

| 格式 | 编码 | 透明质量 | 文件大小 | 编辑软件兼容性 |
|------|------|----------|----------|----------------|
| **WebM** | VP9 | 有损/无损 | 小 | 浏览器好，桌面软件参差 |
| **MOV** | ProRes 4444 | 无损 Alpha | 大 | 最广（剪映、PR、达芬奇均支持） |
| **PNG 序列** | — | 无损 Alpha | 极大 | 万能，但不方便 |

> **注意：MP4 不支持透明底。** H.264/H.265 均不含 Alpha 通道，无法用于透明字幕叠加。

## 三、技术架构

```
┌─────────────────────────────────────────────────────┐
│                    Browser（前端）                   │
│                                                     │
│  ┌──────────┐   ┌──────────┐   ┌─────────────────┐ │
│  │ LRC 解析 │──▶│ 动效调度 │──▶│  Canvas 渲染器  │ │
│  └──────────┘   └──────────┘   └────────┬────────┘ │
│                                          │           │
│  ┌──────────┐   ┌──────────┐            ▼           │
│  │ 参数配置 │   │ 随机引擎 │   ┌─────────────────┐ │
│  └──────────┘   └──────────┘   │ MediaRecorder   │ │
│                                 │ (WebM VP9 Alpha)│ │
│                                 └─────────────────┘ │
└─────────────────────────────────────────────────────┘
```

### 3.1 技术选型

| 模块 | 技术 | 理由 |
|------|------|------|
| 渲染 | Canvas 2D API | 原生支持 Alpha，性能可控 |
| 动画 | GSAP | 时间轴精确控制，缓动丰富 |
| 视频导出 | MediaRecorder API + ffmpeg.wasm | WebM 导出原生支持；MOV 转换通过 ffmpeg.wasm 在浏览器内完成，无需安装 |
| 字体加载 | FontFace API | 本地加载，无跨域问题 |
| 随机引擎 | Seeded PRNG（mulberry32） | 可复现，轻量 |
| 框架 | React + TypeScript | 组件化，类型安全 |

## 四、LRC 解析模块

### 4.1 标准格式

```
[01:23.45]歌词内容
[01:26.00]下一行歌词
```

### 4.2 解析结果数据结构

```ts
interface LyricLine {
  time: number;      // 入场时间（毫秒）
  text: string;      // 歌词文本
  duration: number;  // 持续时长，到下一行的间隔（毫秒）
}
```

### 4.3 特殊情况处理

- **空行 / 间奏段**：触发留白动效（残影消散、粒子飘散等）
- **多行同时戳**：支持合并显示或分层错位显示
- **元数据行**（`[ti:]` `[ar:]` 等）：解析后忽略，不渲染

## 五、动效系统设计

每行歌词独立分配一套动效组合，由随机引擎从以下池中抽取，保证每次生成结果不同。

### 5.1 入场动效（Entrance）

| 名称 | 描述 |
|------|------|
| 打字机 | 字符逐个出现，带光标闪烁 |
| 从左/右/上/下滑入 | 整行或逐字错位滑入 |
| 缩放弹入 | 从小到大，带弹性过冲（elastic easing） |
| 散落聚合 | 字符从随机位置飞向目标位置 |
| 翻转入场 | 绕 X/Y 轴 3D 翻转（矩阵变换模拟） |
| 模糊淡入 | 从高斯模糊逐渐清晰（filter blur） |
| 逐字波浪 | 字符依次以波浪时序出现 |

### 5.2 持续动效（Idle）

| 名称 | 描述 |
|------|------|
| 轻微浮动 | 整行上下缓慢漂移 |
| 字符抖动 | 每个字符独立微幅随机位移 |
| 呼吸缩放 | 整体周期性轻微缩放 |
| 描边脉冲 | 描边宽度周期变化 |
| 静止 | 保持入场结束状态（权重较低） |

### 5.3 出场动效（Exit）

| 名称 | 描述 |
|------|------|
| 淡出 | 透明度渐变至 0 |
| 向上/下飘散 | 移动 + 淡出 |
| 爆散 | 字符向四周飞散 |
| 收缩消失 | 缩小至消失 |
| 残影拖尾 | 留下几帧半透明残影后消失 |

### 5.4 排版随机性

| 维度 | 变化范围 |
|------|----------|
| 水平对齐 | 居中 / 左对齐 / 右对齐 |
| 位置偏移 | 在安全区内随机偏移（±10% 画面宽高） |
| 字间距 | 正常 / 宽松 / 极宽（散排风格） |
| 行内字号差异 | 关键字放大（可选，基于标点或用户标注） |
| 旋转 | 轻微倾斜（±5°）或保持水平 |

### 5.5 随机引擎

使用 **mulberry32**（Seeded PRNG）：

- 同一种子 → 同一动效组合（可复现）
- 不同种子 → 完全不同的结果
- 每行歌词的动效选择相互独立
- 用户可手动输入种子，或点击骰子图标随机生成

## 六、渲染与导出流程

```
1. 用户点击「生成」
2. 初始化 Canvas（指定分辨率，{ alpha: true }）
3. 离线帧渲染模式：以固定 25fps 逐帧驱动 GSAP 时间轴
4. 每帧绘制完成后，MediaRecorder 捕获 Canvas 输出流
5. 全部帧渲染完成，停止录制
6. Blob 转为 .webm 文件，触发浏览器下载
```

### 6.1 离线渲染模式

不依赖实时时钟，而是手动逐帧推进时间轴（`ticker.tick()`），避免因渲染耗时导致掉帧，保证输出帧率稳定。

### 6.2 性能考量

- 使用 `OffscreenCanvas` 在 Worker 中渲染，不阻塞主线程
- 长歌词（> 5 分钟）分段导出，避免内存溢出
- 提供进度条（已渲染帧数 / 总帧数）

## 八、技术风险与对策

| 风险 | 对策 |
|------|------|
| WebM Alpha 在 Safari 不支持 | 页面提示用 ffmpeg 转 MOV；或提供 PNG 序列帧备选导出 |
| 大分辨率 Canvas 性能瓶颈 | 离线帧渲染 + OffscreenCanvas Worker |
| 自定义字体跨域 | FontFace API 本地加载，无跨域问题 |
| 长视频内存溢出 | 分段录制，提示用户用 ffmpeg 合并 |
| 3D 翻转效果 Canvas 2D 局限 | 用矩阵变换模拟透视，或局部引入 Three.js |
| GSAP 离线帧驱动兼容性 | 使用 GSAP 的 `globalTimeline.time()` 手动控制，已有成熟方案 |
