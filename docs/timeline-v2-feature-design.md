# 时间轴精细化编辑 功能设计文档（第二期）

---

## 一、产品背景与目标

**承接上期**
上期时间轴功能为三轨结构（媒体轨道 / 字幕轨道 / 音频轨道）奠定了基础，核心流程已跑通。字幕轨道目前以 LRC 时间戳为唯一数据源，片段只读；媒体片段之间为硬切；字幕无法在轨道上增删。

**本期目标**
在不突破"简洁字幕工具"定位的前提下，解锁三项关键编辑能力，让用户无需反复修改 LRC 原文就能完成常见的时间微调和内容变更：

| 需求 | 解决的痛点 |
|------|-----------|
| 字幕边界拖拽 | LRC 时间戳粒度有限，用户需微调相邻字幕的切换点 |
| 媒体转场特效 | 硬切在某些场景显生硬，需要淡入淡出等柔和过渡 |
| 字幕插入 / 删除 | 歌词外追加说明字幕，或删除不需要的行，无需改 LRC |

**设计边界**
本期不引入自由移动字幕（字幕起始时间保持相对顺序），不支持多音轨，不新增轨道类型。

## 二、功能一：字幕边界拖拽

### 2.1 核心规则：字幕严丝合缝

字幕轨道从本期起进入"无缝模式"：**相邻字幕片段之间不存在间隙**。每条字幕的结束时间 = 下一条字幕的开始时间。这是边界拖拽得以成立的前置约定。

> **数据迁移**：LRC 解析时若存在歌词间隔（gap），用前一个字幕的 end 自动延伸填满，直至下一条字幕的 start。末尾最后一条字幕的 end 保持 LRC 原值（不自动延伸到媒体末尾）。

### 2.2 交互规范

#### 2.2.1 可拖拽边界的识别

- **触发区域**：相邻两个字幕片段之间的 **公共分隔线**，鼠标 hover 热区宽度为 ±4px（总宽 8px）
- **视觉反馈**：
  - 默认态：分隔线为 1px 半透明白色竖线
  - Hover 态：分隔线变为 2px 亮白色，鼠标图标切换为 `col-resize`（←→）
  - 拖拽态：分隔线变为 2px 蓝色，两侧片段实时更新宽度，左片段时长在其内部右下角浮动显示，右片段时长在其内部左下角浮动显示

#### 2.2.2 拖拽行为

设左字幕为 A（start_A, end_A），右字幕为 B（start_B = end_A, end_B），拖拽分隔线：

```
拖拽 Δx 像素（折算为 Δt 秒）后：
  A.end   = clamp(end_A + Δt, start_A + MIN_DURATION, end_B - MIN_DURATION)
  B.start = A.end
  B.end   保持不变
```

- `MIN_DURATION`：单条字幕最小时长 **0.1s**，防止压缩为零
- 拖拽范围：不可超出 A 的 start 或 B 的 end（字幕不会被压没）
- 拖拽时实时更新画布预览区的字幕渲染（当播放头在该范围内时）

#### 2.2.3 首条 / 末条字幕的边界

| 边界位置 | 是否可拖拽 | 说明 |
|---------|-----------|------|
| 首条字幕左边缘 | **不可拖拽** | 首条字幕 start 由 LRC 决定（或插入时确定） |
| 末条字幕右边缘 | **可拖拽** | 单侧拖拽，仅调整末条字幕的 end，不影响其他字幕 |
| 相邻字幕之间 | **可拖拽** | 双向影响 A.end 和 B.start |

#### 2.2.4 操作完成后的数据保存

拖拽 mouseup 时：
1. 将新的 `start` / `end` 写入该字幕的数据对象（覆盖 LRC 原始时间戳）
2. 标记项目状态为"已修改"（后续导出以修改后时间为准）
3. **不**回写 LRC 文本（LRC 文本仍保留原内容，但渲染时以时间轴数据为准）

## 三、功能二：媒体片段转场特效

### 3.1 概念定义

**转场（Transition）**：当媒体轨道中两个相邻片段"首尾相接"时，在连接处叠加一段时间的过渡渲染效果。转场不改变片段时长，而是在两片段交叉区域进行混合绘制。

```
片段 A ────────┤
              ├──── 转场区域（默认 0.5s）────┤
                                            ├──────── 片段 B
```

转场区域从 A 的末尾 0.5s 开始，到 B 的开头 0.5s 结束，总长 1.0s。在此期间同时绘制 A 和 B，按特效类型混合。

### 3.2 转场类型

| ID | 名称 | 视觉描述 | 渲染方式 |
|----|------|---------|---------|
| `none` | 无（硬切）| 前一帧消失，后一帧出现 | 直接切换，无混合 |
| `dissolve` | 混合淡化 | A 逐渐透明，B 逐渐出现 | `globalAlpha` 线性插值：A 从 1→0，B 从 0→1 |
| `black_fade` | 黑幕过渡 | 先淡出到黑，再从黑淡入 | 前半段 A 从 1→0，后半段 B 从 0→1，中间纯黑帧 |

### 3.3 UI 交互：连接处的「+」按钮

#### 3.3.1 触发条件

两个媒体片段在时间轴上**首尾相接**（end_A = start_B，无间隙）时，才在连接处显示转场入口。

#### 3.3.2 外观

```
 [  片段 A  ] ⊕ [  片段 B  ]
              ↑
         转场按钮
```

- 按钮形态：圆形图标，直径 18px，位于两片段连接线中央，垂直居中于媒体轨道
- **无转场（硬切）时**：显示灰色「⊕」（带加号的圆圈）
- **已选转场时**：显示对应颜色的图标（混合淡化：蓝色；黑幕过渡：深灰色）
- Hover 时按钮放大至 22px，显示 tooltip（当前特效名称）
- 轨道缩放过小导致按钮无法显示时（连接处宽度 < 16px），隐藏按钮，改为 hover 连接线高亮提示

#### 3.3.3 点击行为

点击「⊕」按钮，弹出**转场选择 Popover**（浮层，非模态弹窗）：

```
┌─────────────────────────┐
│  选择转场效果            │
├─────────────────────────┤
│  ◎ 无（硬切）           │  ← 当前选中项用单选圆点标注
│  ○ 混合淡化             │
│  ○ 黑幕过渡             │
├─────────────────────────┤
│  时长：[  0.5s  ] ↕     │  ← 数字输入框，支持 0.1s–2.0s
└─────────────────────────┘
```

- 选择特效后**立即生效**，无需确认按钮，点击 Popover 外部关闭
- 时长调整范围：0.1s – 2.0s，步进 0.1s；默认 0.5s
- 转场时长不得超过相邻两片段中较短那个的一半，超出时自动 clamp 并在 UI 中提示

### 3.4 渲染逻辑

每帧绘制时，判断当前时间是否落在某个转场区域内：

```
转场区域：[end_A - duration/2, start_B + duration/2]
当前帧时间 t 在区间内时：

  progress = (t - (end_A - duration/2)) / duration  // 0.0 → 1.0

  dissolve:
    drawImage(frameA)  alpha = 1 - progress
    drawImage(frameB)  alpha = progress

  black_fade:
    if progress < 0.5:
      drawImage(frameA)  alpha = 1 - progress*2
    else:
      drawImage(frameB)  alpha = (progress - 0.5)*2
    // 下层已有黑色背景（纯色兜底层），无需额外绘制黑帧
```

### 3.5 导出兼容性

转场在逐帧导出时按上述渲染逻辑执行，不需要额外的合成步骤。`none` 转场导出行为与现有无变化。

## 四、功能三：字幕插入与删除

### 4.1 字幕删除

#### 4.1.1 操作入口

| 入口 | 触发条件 |
|------|---------|
| 键盘 `Delete` | 字幕轨道中有字幕片段被选中时 |
| 右键菜单 → 「删除字幕」 | 右键点击字幕片段 |

#### 4.1.2 删除规则

被删除字幕留下的时间空档，由**前一条字幕延伸**来填满（保持严丝合缝）：

```
删除前：[A: 0–3s] [B: 3–5s] [C: 5–8s]
删除 B：[A: 0–5s]            [C: 5–8s]
  → A.end 延伸到原 B.end = 5s
```

**边界情况：删除首条字幕**

没有前一条字幕可延伸，改为**后一条字幕提前**来填满：

```
删除前：[A: 0–3s] [B: 3–5s] [C: 5–8s]
删除 A：           [B: 0–5s] [C: 5–8s]
  → B.start 提前到原 A.start = 0s，其余字幕 start/end 不变
```

**边界情况：删除唯一一条字幕**

字幕轨道变为空，允许存在（空轨道状态）。

#### 4.1.3 操作反馈

删除后：
- 自动选中填补字幕（前一条，或首条被删时为新首条）
- 右侧面板同步刷新至选中字幕的编辑内容
- 支持 `Ctrl+Z` 撤销

### 4.2 字幕插入

#### 4.2.1 操作入口

字幕轨道标签右侧新增「**+**」按钮（与媒体轨道「+」按钮同款样式），点击后在**当前选中字幕的后面**插入一条新字幕。

- 若当前无选中字幕：「+」按钮不可用（灰色禁用态），hover 时 tooltip 提示「请先选中一条字幕」
- 若轨道为空：「+」按钮可用，在时间 0 处插入首条字幕

#### 4.2.2 新字幕的默认属性

| 属性 | 默认值 |
|------|--------|
| 文本内容 | `新字幕` |
| 时长 | 1.0s |
| 插入位置 | 紧接选中字幕之后（start = 选中字幕的 end） |
| 样式 | 继承当前字幕样式全局配置 |

#### 4.2.3 插入对后续字幕的影响

插入操作会将**被插入字幕之后的所有字幕**整体后移 1.0s（严丝合缝规则，不产生间隙）：

```
插入前：[A: 0–3s] [B: 3–6s] [C: 6–9s]
在 A 后插入（1.0s）：
插入后：[A: 0–3s] [NEW: 3–4s] [B: 4–7s] [C: 7–10s]
  → B、C 的 start/end 各加 1.0s
```

音频轨道和媒体轨道**不受影响**（总时长自动因字幕末尾延长而更新）。

#### 4.2.4 插入后的 UX 流程

1. 新字幕片段出现在轨道上，自动高亮选中
2. 右侧编辑面板立即切换到该字幕的编辑视图，文本框内容 `新字幕` 全选，用户可直接键入
3. 时间轴视口自动滚动，确保新字幕片段在可见区域内
4. 支持 `Ctrl+Z` 撤销整个插入操作（包括后续字幕位移）

## 五、数据模型变更

### 5.1 字幕片段数据结构

```typescript
interface CaptionSegment {
  id: string
  text: string
  start: number          // 秒，本期起可由用户修改
  end: number            // 秒，本期起可由用户修改
  source: 'lrc' | 'manual'  // 'lrc'=来自解析，'manual'=用户插入
  style?: CaptionStyleOverride  // 逐行样式覆盖（已有字段）
}
```

新增 `source` 字段用于调试和未来功能区分，不影响渲染逻辑。

### 5.2 转场数据结构

```typescript
interface MediaTransition {
  type: 'none' | 'dissolve' | 'black_fade'
  duration: number  // 秒，默认 0.5
}

// 存储位置：MediaSegment 新增 transitionIn 字段
interface MediaSegment {
  // ... 已有字段 ...
  transitionIn?: MediaTransition  // 与前一片段之间的转场（第一片段无此字段）
}
```

## 六、实现优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **P0** | 字幕无缝模式数据迁移 | LRC gap 自动延伸，所有功能的前提 |
| **P0** | 字幕边界拖拽交互 | 核心 UX，鼠标事件 + 实时宽度更新 |
| **P1** | 字幕删除（Delete 键） | 交互简单，规则清晰 |
| **P1** | 字幕插入（「+」按钮）| 需处理后续字幕位移 |
| **P1** | 转场 UI（「⊕」按钮 + Popover）| UI 部分 |
| **P2** | 转场渲染（dissolve）| 渲染逻辑，需改帧绘制管线 |
| **P2** | 转场渲染（black_fade） | 渲染逻辑 |
| **P3** | Ctrl+Z 撤销（字幕增删 + 拖拽）| 需引入 undo stack |

## 七、边界情况汇总

| 场景 | 处理方式 |
|------|---------|
| 拖拽边界时左字幕被压缩到 < 0.1s | clamp，不允许继续拖动 |
| 拖拽边界时右字幕被压缩到 < 0.1s | clamp，不允许继续拖动 |
| 转场时长 > 相邻片段较短那个的 50% | 自动 clamp，Popover 内提示「时长已自动限制」 |
| 媒体片段之间有间隙（非相邻）| 不显示转场「⊕」按钮 |
| 删除首条字幕 | 后续字幕不整体平移，仅首条被删，后一条 start 提前到 0 |
| 删除唯一字幕 | 字幕轨道置空，允许 |
| 在末尾字幕后插入 | 1.0s 追加到末尾，不影响任何已有字幕 |
| 插入后总时长超出媒体 / 音频末尾 | 正常，总时长 = max(各轨道末尾)，允许字幕轨道最长 |
| 字幕轨道为空时按 Delete | 无响应 |

## 八、暂不纳入的方向（留备忘）

- **字幕自由拖动起始位置**：本期仅支持边界拖拽，不支持整条字幕在时间轴上任意移动（会打破严丝合缝规则）
- **转场参数化（曲线、方向）**：混合淡化和黑幕过渡均为线性，不支持缓动曲线，留专业版
- **音频淡入淡出**：音频轨道转场独立于媒体转场，暂不支持
- **字幕拆分**：选中字幕从播放头位置一分为二，复杂度较高，留后期
- **多选字幕批量操作**：目前仅支持单选
