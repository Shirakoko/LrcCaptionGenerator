# LrcCaptionGenerator 逐行定制化进阶功能方案

## 一、背景与目标

### 现状
当前工具以随机数种子驱动全局生成：每行歌词的位置（X/Y/对齐方式/字号/旋转角度）和特效组合（入场/持续/退场）均由 `Prng` 随机决定，用户只能通过换种子来影响结果，缺少精细控制手段。

### 目标
在不破坏现有随机生成流程的前提下，允许用户对**任意一行歌词**单独覆盖以下参数：

| 类别 | 可编辑项 |
|------|---------|
| 布局 | 起始 X 位置、Y 位置、字号、对齐方式、字间距、旋转角度 |
| 特效 | 入场动画、持续动画、退场动画 |

编辑后的行与未编辑行共存于同一时间轴，可实时预览并随时撤销（恢复随机生成值）。

## 二、核心数据结构设计

### 2.1 逐行覆盖表（LineOverride）

```typescript
// src/types/override.ts（新增文件）

export interface LayoutOverride {
  x?: number;               // 行锚点 X（画布像素）
  y?: number;               // 行锚点 Y（画布像素）
  fontSize?: number;        // 字号（px）
  align?: 'left' | 'center' | 'right';
  letterSpacingExtra?: number;
  rotation?: number;        // 度数
}

export interface EffectOverride {
  entrance?: EntranceName;
  idle?: IdleName;
  exit?: ExitName;
}

export interface LineOverride {
  layout?: LayoutOverride;
  effects?: EffectOverride;
}

// 整个场景的覆盖表，key = 歌词行索引（0-based）
export type OverrideMap = Record<number, LineOverride>;
```

**设计要点：**
- 字段全部可选（`?`），缺失字段回退到随机生成值
- 覆盖表与 `SceneOptions` 解耦，独立管理，方便序列化/持久化
- 索引以歌词行号为键，与 LRC 解析顺序严格对应

### 2.2 SceneController 接口扩展

```typescript
// 在现有 SceneOptions 中增加覆盖表参数
export interface SceneOptions {
  seed?: number | string;
  config: RenderConfig;
  overrides?: OverrideMap;   // ← 新增
}
```

`buildLineState()` 在随机生成布局参数后，优先取 `override.layout` 中的值覆盖对应字段；`pickEffects()` 同理，被覆盖的特效字段跳过随机选取直接使用指定值。

点击某行的 [编辑] 后，该行下方展开：

```
┌──────────────────────────────────────────┐

│  [应用并预览]          [还原随机值]       │

```

**控件说明：**

| 参数 | 控件类型 | 范围/选项 |
|------|---------|---------|
| 位置 X | 数字输入 + 拖拽滑块 | 0 ~ 画布宽度 |
| 位置 Y | 数字输入 + 拖拽滑块 | 0 ~ 画布高度 |
| 字号 | 数字输入 + 滑块 | 24 ~ 200 px |
| 对齐方式 | 三态按钮 | left / center / right |
| 字间距 | 滑块 + 数字 | −4 ~ 16 px |
| 旋转 | 滑块 + 数字 | −20 ~ 20 ° |
| 入场动画 | 下拉菜单 | 全部 10 种 + "随机" |
| 持续动画 | 下拉菜单 | 全部 4 种 + "随机" |
| 退场动画 | 下拉菜单 | 全部 6 种 + "随机" |

下拉菜单中"随机"选项表示不设覆盖，回归种子生成结果。

### 3.4 画布直接拖拽（可选进阶）

在预览画布上，当某行文本处于可见状态时，支持点选并拖拽来直接设定 X/Y，拖拽结果同步回行编辑面板的滑块和数字框。实现方式：

1. 预览 canvas 叠加一层透明 `<div>` 接收鼠标事件
2. 建立像素坐标 → 行归属的命中检测（基于 `LineLayout` 的包围盒）
3. `pointerdown` → 记录命中行 → `pointermove` 更新 override → `pointerup` 提交

## 四、交互流程

### 4.1 首次 Build

```
用户点击 Build Preview
  → parseLRC()  →  SceneController.build(lyrics, { seed, config, overrides: {} })
  → 所有行使用纯随机生成
  → 显示行列表，所有行无覆盖标记
```

### 4.2 编辑单行

```
用户展开第 N 行的编辑面板
  → 读取该行的当前随机生成值（作为默认显示值）
  → 用户调整参数
  → 点击 [应用并预览]
      → 更新 overrideMap[N]
      → 仅重建第 N 行的 LineState（不重建整个场景）
      → 用新的 LineState 替换 SceneController.activeLines[N]
      → 重建该行对应的 GSAP 子时间轴，插回 masterTl
      → 刷新预览
```

> **重要：** 局部重建（仅重建单行动画）而非全量重建，保证其他行的动画状态不受干扰。

### 4.3 还原随机值

```
用户点击 [还原随机值]
  → 删除 overrideMap[N]
  → 以原始种子重新生成第 N 行
  → 重建该行子时间轴
  → 编辑面板内各控件恢复为随机生成值的显示
```

### 4.4 重新 Build（换种子）

```
用户修改种子后点击 Build Preview
  → 保留已有的 overrideMap（用户的手动编辑不丢失）
  → 重建所有行：无覆盖的行使用新种子随机，有覆盖的行仍以覆盖值为准
```

## 五、SceneController 内部改动

### 5.1 子时间轴化

当前所有行的动画均直接插入 `masterTl`。为支持局部重建，需要将每行的动画改为**独立的子时间轴（child timeline）**，以行索引为 `id` 挂载到 master：

```typescript
// 当前（伪代码）
masterTl.add( entranceTween, startSec );
masterTl.add( idleTween, startSec + 0.6 );
masterTl.add( exitTween, exitSec );

// 改造后
const lineTl = gsap.timeline({ id: `line-${i}` });
lineTl.add( entranceTween, 0 );
lineTl.add( idleTween, 0.6 );
lineTl.add( exitTween, durSec - 0.5 );
masterTl.add( lineTl, startSec );   // 整个子 tl 插入 master 的指定时间点
```


## 六、覆盖状态的导出与导入（持久化）

### 6.1 导出格式

用户编辑完成后，可将 `overrideMap` 与当前种子/LRC 一起导出为 JSON 存档：

```json
{
  "seed": 1234567890,
  "overrides": {
    "2": {
      "layout": { "x": 960, "y": 400, "fontSize": 80 },
      "effects": { "entrance": "wave", "exit": "explode" }
    },
    "5": {
      "effects": { "idle": "none" }
    }
  }
}
```

### 6.2 导入

UI 提供"导出配置"和"导入配置"按钮（JSON 文件），下次打开同一 LRC 时可恢复编辑状态。

## 八、关键技术风险与对策

| 风险 | 描述 | 对策 |
|------|------|------|
| GSAP 子时间轴替换 | masterTl 动态替换子 tl 会影响全局 duration 和播放头 | 替换前 pause，替换后重设 time，测试多行场景 |
| 局部重建与种子状态 | 重建单行时需独立重放该行的 Prng 序列（跳过随机步骤） | 每行 Prng 独立实例化（当前已如此），覆盖字段直接绕过 rng.call |
| X/Y 坐标体系 | 当前 X 由对齐+字宽自动推导，直接给 anchorX 会打破对齐逻辑 | 覆盖 X 时强制 align 为 'left'（从锚点开始渲染），或将 anchorX 解释为 line 中心 X |
| 字号自动缩放 | 字号过大时 layout.ts 会自动缩减，覆盖字号后仍需保留此机制 | 覆盖 fontSize 作为"期望值"，仍经过缩放逻辑处理后写入 layout.fontSize |
| 行列表性能 | 歌词行数多（>100 行）时列表渲染卡顿 | 使用虚拟列表（仅渲染可视区域内的行） |
